# 算数教材ジェネレータ 作業ガイド

## 適用範囲

このファイルはリポジトリ全体に適用する。
配下に別の `AGENTS.md` がある場合は、そのディレクトリでは配下の指示も優先して適用する。

## 作業前に確認するもの

- リポジトリ全体の構成と実行方法は `README.md` を確認する。
- ディレクトリ設計を変える場合は `docs/directory-structure-proposal.md` を確認する。
- 教材を変更する場合は、対象の `generators/<generator-slug>/SPEC.md` を先に読む。
- 作業開始時に `git status --short` を確認し、既存の未コミット変更を上書きしない。

## 実装ルール

- 教材固有の画面、作問処理、検証処理、描画、スタイル、テスト、仕様書は、原則として同じ `generators/<generator-slug>/` 内に置く。
- 既存ジェネレータのディレクトリを丸ごと複製せず、必要なファイルだけを作る。
- 2つ以上の教材で同じ実装が実際に必要になるまで、ルートに `lib/`、`shared/`、共通CSSなどを作らない。
- 教材固有のルールを、見かけ上の重複だけを理由に共通化しない。
- 仕様を変更した場合は、`SPEC.md`、実装、テスト、必要に応じて画面上の説明を同じ作業で同期する。
- 乱数を使う作問処理は、可能な限り再現可能な入力またはseedでテストできる構造を保つ。
- 問題の生成条件だけでなく、解の成立、一意性、難易度条件など、`SPEC.md` が保証するとしている性質をテストする。
- 空の将来用ファイルや、使用されない設定項目を追加しない。

## UI・印刷物の確認

- UIや描画を変更した場合は、ローカルサーバー経由で対象ジェネレータを表示して確認する。
- 画面表示だけでなく、印刷プレビューまたは印刷向けレイアウトも確認する。
- 問題用紙と解答・解説がある場合は、双方について欠落、重なり、はみ出し、不自然な改ページがないか確認する。
- PDFを作成する作業では、グローバルガイドの `guides/pdf-creation-howto.md`（「読ませるPDFを作る How-to」）に従い、全ページを検査する。

## 検証コマンド

変更範囲に応じて、少なくとも関連する個別テストを実行する。作業完了前には原則として全体テストも実行する。

```bash
npm test
```

`kazu-sagashi` の生成品質や多数seedの確認に関係する変更では、コーパステストも実行する。

```bash
npm run test:kazu-sagashi:corpus
```

ローカル表示の確認には次を使用する。

```bash
npm run serve
```

テストを実行できなかった場合や、ブラウザ・印刷確認を省略した場合は、完了報告で理由と未確認範囲を明記する。

## 変更時の注意

- 依頼範囲と無関係なファイルを整形・改名・削除しない。
- 既存の未コミット変更が対象ファイルと重なる場合は、内容を確認して保持する。
- 削除や大きな構成変更では、参照元、import、HTMLのパス、ルート一覧、テスト入口、ドキュメントのリンクを確認する。
- 新しいジェネレータを追加した場合は、ルートの `index.html` から到達できるようにし、`README.md` の収録一覧も更新する。

## 完了条件

- 実装が対象の `SPEC.md` と一致している。
- 関連テストと `npm test` が成功している、または未実行理由が明記されている。
- UI変更はブラウザで、印刷物変更は印刷レイアウトで確認されている。
- 仕様、画面説明、テスト、リンクに取り残された古い記述がない。
- 完了報告に、変更内容、確認内容、残る制約や未確認事項を簡潔に記載している。
