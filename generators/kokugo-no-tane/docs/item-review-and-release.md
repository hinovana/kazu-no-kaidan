# こくごのたね 作問見直し・公開判断仕様書

| 項目 | 値 |
| --- | --- |
| 文書状態 | 検討中 |
| 文書版 | `item-review-release.v0.1-draft` |
| 開発形態 | 個人開発 |
| 最終更新日 | 2026-07-14 |
| 実装状態 | 未実装 |
| 運用状態 | 未運用 |
| 現段階の実装範囲 | 見直し記録・状態遷移のプロトタイプまで。校正済みとしての公開はまだ行わない |
| 主な未決事項 | 見直し画面、外部の大人へ試してもらう方法、公開取り下げ手順 |

## 1. 目的

本書は、自動生成した問題をどの状態で利用できるか、開発者本人が何を確認するか、修正後にどの確認をやり直すかを定める。

生成系が本文、設問、正答、根拠、品質点を同時に作る場合、それらが同じ誤解を共有する可能性がある。自動検査の合格を、正しさの証明または公開可否の最終判断として扱わない。

## 2. 利用区分

### 2.1 未校閲の練習素材

ブラウザで生成した直後の問題は、機械検査に合格していても人間未確認である。現行プロトタイプでは `development_preview` とし、最初の人間レビューが終わるまでは児童に使用しない。レビュー後に開発者本人が練習利用可能と判断した版だけを `unreviewed_practice` へ進め、次の表示を伴う場合に限り日常練習へ使用できる。

```text
この問題は自動生成された未校閲の練習素材です。
採点結果は学力判定や成績評価には使用できません。
```

未校閲の練習素材は、成績、選抜、学力診断、教材品質の実証、校正済み問題集として利用しない。

### 2.2 見直し済み練習問題

作問チェック、答えを隠した解き直し、表現・公平性確認を終えた問題は `reviewed_practice` とする。児童試行前のため、実測難易度や識別力を保証しない。

### 2.3 校正済み評価問題

児童試行と校正を完了し、開発者本人が公開可能と判断した固定問題だけを `release_ready` とする。動的に再生成した別問題へ、元問題の公開状態や難易度を引き継がない。

## 3. 状態遷移

### 3.1 正常系

```text
generated
  ↓ 自動検査合格
automated_checks_passed
  ↓ 作問チェック
self_reviewed
  ↓ 正答・根拠を隠した別セッションで解き直す
blind_recheck_passed
  ↓ 表現・公平性・アクセシビリティ確認
fairness_checked
  ↓ 試行条件を確認
pilot_ready
  ↓ 児童試行と分析
calibrated
  ↓ 公開条件を最終確認
release_ready
```

`automated_checks_passed` は構造・来歴・表記・テンプレート契約の検査通過を表し、文章の自然さや教育的妥当性を保証しない。現行プロトタイプでは `development_preview` として画面へ出し、最初の人間レビュー後に開発者本人が判断したものだけを `unreviewed_practice` へ進める。`generated` のままの問題は画面へ出さない。自動検査で正答根拠契約、表記、ふりがな等の不整合を検出した問題は破棄する。

### 3.2 差し戻し・却下

どの状態からも次へ遷移できる。

| 状態 | 意味 |
| --- | --- |
| `needs_revision` | 修正後に必要な確認をやり直す |
| `rejected` | 当該item IDでは採用しない |
| `withdrawn` | 公開後に問題が見つかり利用を停止した |

`needs_revision` から直接元の状態へ戻さない。変更内容に応じて、後述する失効点まで状態を戻す。

## 4. 個人開発で確認を分ける方法

開発者本人が、**情報と作業セッションを分けて**確認する。

| 確認モード | 行うこと | 混同を防ぐ条件 |
| --- | --- | --- |
| 生成 | 生成条件、出力、版情報を固定する | 正答・根拠を含む完全データを保存する |
| 作問チェック | 測定設計、文章、設問、採点基準を確認する | チェック結果と修正内容を記録する |
| 答えを隠した解き直し | 児童向け本文と設問だけで解く | 正答・根拠・採点表を閉じ、別セッションで行う |
| 表現・公平性チェック | 題材、アクセシビリティ、ふりがな、印刷表示を確認する | 作問内容ではなく専用チェックリストに集中する |
| 校正 | 試行条件、回答データ、統計値を確認する | 自動品質点を採否根拠にしない |
| 公開判断 | 利用区分と公開範囲を確認する | 直前の編集セッションと分け、全チェック履歴を見る |

答えを隠した解き直しは、可能なら時間を空けて行う。別の大人に解いてもらえる場合は追加の外部確認として記録する。開発者本人による解き直しを、第三者による独立検証とは表示しない。

## 5. 自動検査ゲート

自動検査は、少なくとも次を確認する。

- 本文、設問、正答、根拠IDの参照整合性
- 抜き出し条件と候補数
- 選択問題の正答数
- 採点要素の存在
- 学年別語彙と漢字表記
- 初出ふりがなと再出省略
- 禁止題材・禁止表現
- 参照アンカーとの表層類似
- 必須の生成来歴情報

生成系による自己解答、反証生成、品質スコアは、自動検査の一部として利用できる。ただし、それだけで `self_reviewed` 以降へ遷移させない。

## 6. 作問チェック

開発者本人が、次を1項目ずつ `pass|revise|reject|not_applicable` で記録する。

### 6.1 測定設計

- [item-blueprint.md](item-blueprint.md) の出題設計IDと一致する。
- 各問の主たる測定能力が1つに定まっている。
- 心情理解と心情推論、明示情報探索と人物像推論を混同していない。
- 本文外の常識や望ましい行動を答えさせていない。
- 記述力や語彙力が読解力へ過度に混入していない。

### 6.2 本文

- 時系列、因果、人物関係に矛盾がない。
- 必要な状況が本文内で説明されている。
- 対象学年の生活経験から大きく外れていない。
- 結末や心情変化が唐突ではない。
- 参照問題の文章、人物、固有の出来事、展開を模倣していない。

### 6.3 設問と採点

- 本文だけで解答できる。
- 正答根拠を特定できる。
- 抜き出し・選択問題の正答が一意である。
- 記述問題の必須要素、部分点、許容例、誤答条件が明確である。
- 設問同士が答えを教えていない。

## 7. 答えを隠した解き直し

新しい確認セッションを開始し、児童が見る本文、設問、選択肢、表示条件だけを開く。次は見ない。

- 模範解答
- 正答根拠ID
- 採点要素
- 生成時の意図
- 自動品質スコア

解き終わった後で正本の解答を開き、自分の解答と根拠箇所を比較して記録する。別の大人に解いてもらった場合も、同じ形式で記録する。

次の場合は合格させない。

- 正本と異なるが本文上妥当な答えが出た。
- 正本と同じ答えでも、異なる根拠で成立した。
- 設問の意味を複数に解釈できた。
- 指定字数や回答形式が、正答の一意性を人工的に作っているだけだった。

## 8. 表現・公平性・アクセシビリティ確認

### 8.1 公平性

- 特定の家族構成を当然視していない。
- 所有物、旅行、習い事など経済条件を正答の前提にしていない。
- 都市・地方、文化、宗教、国籍による経験差を正答の前提にしていない。
- 性別と性格、役割、能力を固定していない。
- 障害、病気、発達特性を否定的な装置として扱っていない。
- 読解対象以外の専門知識を要求していない。

### 8.2 アクセシビリティ

- ふりがなが親文字と対応し、本文の視認性を損なっていない。
- 字の大きさ、行間、余白、選択肢の区別が対象学年に適している。
- 色だけで正答欄や重要情報を区別していない。
- 問題用紙と解答用紙で、欠落、重なり、はみ出し、不自然な改ページがない。
- 読み上げや代替回答を使う場合、何を測定対象外にしたかを記録している。

## 9. 修正時に戻す状態

| 変更内容 | 戻す状態 | 理由 |
| --- | --- | --- |
| 誤字のみで意味・読み・レイアウトに影響しない | `self_reviewed` | 答えを隠した解き直し以降を再確認する |
| 本文の語句、出来事、心情表現 | `automated_checks_passed` | 正答根拠と難度が変わり得る |
| 設問、選択肢、正答、採点基準 | `automated_checks_passed` | 正答一意性と測定能力が変わる |
| 漢字、ふりがな、改ページ、文字サイズ | `self_reviewed` | 読みやすさと実測難易度が変わり得る |
| 出題設計ID、学年、生成難度 | `generated` | 別問題として扱う |
| 校正済み問題の表示条件 | `fairness_checked` | 試行条件との同等性を確認する |

修正後は新しい `item_revision` を発行し、旧版の見直し履歴を上書きしない。校正の失効条件は [calibration-and-fairness.md](calibration-and-fairness.md) に従う。

## 10. 却下理由

却下または差し戻し時は、少なくとも1つの理由コードと説明を記録する。

| コード | 意味 |
| --- | --- |
| `NO_UNIQUE_ANSWER` | 正答が一意でない |
| `NO_TEXT_EVIDENCE` | 本文上の根拠が不足している |
| `CONSTRUCT_MISMATCH` | 出題設計と測定能力が一致しない |
| `GRADE_MISMATCH` | 語彙、表記、経験が対象学年に合わない |
| `UNCONTROLLED_BACKGROUND_KNOWLEDGE` | 背景知識が正答を左右する |
| `UNFAIR_OR_STEREOTYPED` | 公平性または表現上の問題がある |
| `ACCESSIBILITY_FAILURE` | 表示、ふりがな、回答手段に問題がある |
| `TOO_SIMILAR_TO_REFERENCE` | 参照問題の表層を模倣している |
| `CALIBRATION_FAILURE` | 試行結果が採否基準を満たさない |
| `RIGHTS_OR_PRIVACY_FAILURE` | 権利または個人情報上の問題がある |

## 11. 見直し記録

問題ごとに次を保持する。

```json
{
  "item_id": "kt-item-000001",
  "item_revision": 3,
  "status": "blind_recheck_passed",
  "check_history": [
    {
      "stage": "self_reviewed",
      "check_session_id": "check-session-001",
      "checked_at": "2026-07-14T00:00:00+09:00",
      "result": "pass",
      "reason_codes": [],
      "notes": ""
    }
  ],
  "supersedes_revision": 2
}
```

個人名や人物IDは不要である。代わりに、編集・解き直し・校正をいつ分けて行ったか追跡できるセッションIDを保持する。

## 12. 公開ゲート

`release_ready` にできるのは、開発者本人が次をすべて確認した固定版だけである。

- 出題設計が確定している。
- 必須の自動検査がすべて合格している。
- 作問チェック、答えを隠した解き直し、公平性確認が完了している。
- 児童試行と校正が [calibration-and-fairness.md](calibration-and-fairness.md) の基準を満たしている。
- 出典、参照アンカー、生成来歴、編集履歴を追跡できる。
- 最終編集とは別のセッションで、公開条件と見直し履歴を確認している。
- 利用範囲、対象学年、校正条件、失効条件が表示されている。

公開可能という判断は、同じitem IDの同じrevisionにだけ有効である。seedや条件が同じでも再生成した内容が異なる場合は引き継がない。
