## 1. 目的

本書は、国語読解問題ジェネレータ「こくごのたね」で使用する、次のデータベースの作成・更新・検証方法を定める。

1. 小学校低学年・高学年の語彙候補データベース
2. 小学1〜3年生向け漢字データベース
3. 漢字表記、読み、初出ふりがなを結び付ける表記データベース

根拠資料から得られた事実と、本プロジェクトが教育目的で決めた推奨値を分離し、各データが「どの資料の、どの版を、どの規則で加工して作られたか」を追跡可能にする。

## 2. 学年と生成難度の分離

画面では `学年`、`生成難度プロファイル`、`本文長` を独立して選択する。生成難度は生成条件であり、児童にとっての実測難易度ではない。

| 軸 | 選択肢 | 制御対象 |
| --- | --- | --- |
| 学年 | 小学1年・2年・3年 | 語彙、既習漢字、読み、ふりがな、文法表現、題材の生活経験 |
| 生成難度プロファイル | 1〜5 | 現行プロトタイプでは各設問セットの心情記述問題の根拠間距離 |
| 本文長 | 短め・ふつう・長め | 全角空白とふりがなの読みを除いた本文文字数帯 |

学年が上がることを生成難度上昇と同一視しない。

例：`小学1年・生成難度5` は、小学1年生に理解可能な語彙と読み支援を使いながら、根拠を複数箇所へ分散した問題とする。`小学3年・生成難度1` は、小学3年生相当の自然な表記を使いながら、短く明示的な問題とする。

```ts
type GenerationProfile = {
  grade: 1 | 2 | 3;
  generationProfile: 1 | 2 | 3 | 4 | 5;
  curriculumPoint?: "start" | "summer" | "end";
  databaseRelease: string;
};
```

出題設計は [item-blueprint.md](item-blueprint.md)、実測難易度の付与は [calibration-and-fairness.md](calibration-and-fairness.md) を正本とする。本DBは、生成難度や実測難易度を語彙学年へ混入させない。

## 3. 根拠資料の階層

資料の役割と証拠強度を混同しない。

| 証拠区分 | 意味 | 使用例 |
| --- | --- | --- |
| A: 公的規範 | 学年配当や教科書表記を直接規定する | 文部科学省の学年別漢字配当表、教科書検定基準 |
| B: 公開研究データ | 語の出現、頻度、品詞、教育基本語彙区分を示す | 国立国語研究所の教育基本語彙、研究版教科書語彙リスト |
| C: 本プロジェクト導出値 | Bのデータから再現可能な規則で算出する | 初出学年、学年別頻度、推奨学年候補 |
| D: 専門家判断 | 数値だけで決められない教育的判断 | 多義語、生活経験、安全性、児童に自然な語か |

漢字の `allocation_grade` はAとする。語彙の `first_observed_grade` はB、`recommended_grade` はCまたはDとする。

## 4. 採用する一次資料

### 4.1 漢字

#### K1. 小学校学習指導要領の学年別漢字配当表

- 提供者：文部科学省
- 用途：漢字の配当学年を確定する正本
- 対象件数：第1学年80字、第2学年160字、第3学年200字
- 累積件数：第1学年80字、第2学年まで240字、第3学年まで440字
- 参照： [小学校学習指導要領（平成29年告示）解説 国語編](https://www.mext.go.jp/content/20220606-mxt_kyoiku02-100002607_002.pdf)
- 版違いの補助確認： [文部科学省掲載の平成20年告示版・学年別漢字配当表](https://www.mext.go.jp/a_menu/shotou/cs/1320015.htm)

平成29年告示の告示PDFと国語編解説PDFから独立に転記して突合する。公式HTMLは平成20年告示版であるため、第1〜3学年の字面が一致しても平成29年告示の独立抽出経路には数えない。字体差があり得るため、Unicodeコードポイントだけでなく、出典上の表記と正規化後表記を保持する。AIによる突合を人手確認済みとして記録せず、人間が原典と照合するまでは `manual_check_status=pending` とする。

#### K2. 音訓の小・中・高等学校段階別割り振り表

- 提供者：文部科学省
- 用途：漢字単体ではなく、学校段階で扱う読みの候補を確認する
- 参照： [音訓の小・中・高等学校段階別割り振り表](https://www.mext.go.jp/a_menu/shotou/new-cs/1385768.htm)

この表は学校段階の目安であり、小学1・2・3年の読みを一意に確定するものではない。読みの学年は、語彙側の用例・表記と組み合わせて決定する。

#### K3. 義務教育諸学校教科用図書検定基準

- 提供者：文部科学省
- 用途：当該学年より後に配当される漢字を使用する場合の読み支援ルール
- 参照： [義務教育諸学校教科用図書検定基準](https://www.mext.go.jp/a_menu/shotou/kyoukasho/kentei/1343945.htm)

本プロジェクトでは、同基準にある「少なくとも単元ごとの初出で読み方を示す」という考え方を、問題本文の初出ふりがな規則へ適用する。

### 4.2 語彙

#### V1. 教育基本語彙データベース 2009年版B

- 提供者：国立国語研究所
- 用途：教育上の基本語彙候補、見出し、表記、品詞、分類、複数語彙表の採否情報
- 収録規模：27,234語
- 配当区分：小学校低学年6,865語、小学校高学年8,620語、中学校11,749語
- 参照： [教育基本語彙データベース](https://mmsrv.ninjal.ac.jp/brfvep/)
- 使用ファイル：`kyoikukihongoi_2009B.csv`
- 原本文字コード：CP932

低学年区分は小学1〜3年、高学年区分は小学4〜6年をまとめた区分である。これだけで個別学年へ分割してはならず、国が定めた公式語彙配当とも表示しない。

#### V2. WEB版小学校教科書語彙リスト

- 作成者：山本裕子・鷲見幸美・川村よし子
- 用途：学年・教科別の語彙出現頻度、見出し、読み、品詞、複合語情報
- 規模：約2万語
- 特徴：小学校の全教科・全学年について、語と複合語の頻度を検索できる
- 参照： [WEB版小学校教科書語彙リスト](https://nihongo-de.com/shogakugoi/)
- 解説： [小学校教科書語彙リストについて](https://minio2.overworks.jp/doutor/shogakugoi/public/shogakugoi.pdf)

このリストから得られるのは「ある教科書で、その学年に出現した」という観測事実であり、「その学年なら全児童が理解している」という保証ではない。分析対象は各教科・各学年につき1種類の教科書であるため、出版社差も残る。

#### V3. 現代書き言葉UniDic

- 提供者：国立国語研究所
- 用途：語彙素、品詞、読み、活用、語の同一性を正規化する
- 参照： [UniDic 最新版ダウンロード](https://clrd.ninjal.ac.jp/unidic/download.html)
- 採用版：実装開始時に固定し、DBリリース情報へ記録する
- ライセンス選択：修正BSD

UniDicの解析結果を無検査で正解にしない。児童向け表記、固有名詞、複合語、読み分けは開発者本人の目視確認対象とする。

### 6.8 実装済み候補版の語彙レコード

`vocabulary-db.v0.1-ninjal-2009b-candidate` は、V1の原項目を失わずに取り込むための候補版である。最終的な `lexeme`、`lexeme_form`、`lexeme_grade_policy` へ統合する前段階として、次を保持する。

- 安定 `lexeme_id` と原資料の通し番号
- `lower_elementary_1_3` または `upper_elementary_4_6`
- 見出し、読み候補、表記原欄、品詞
- 7種類の教育基本語彙表における採否値と出現数
- 語種と分類番号
- CSV行まで戻れる出典位置
- `candidate_unreviewed`、`manual_check_status=pending`
- `active_for_generation=false`

原CSVの「表記」欄には漢字表記だけでなく語源表記なども含まれるため、候補版では `notation` として保存し、児童向け表記へ直接使用しない。見出しは `reading_candidate` に複写するが、語義分割、読み、活用、表記の確定値とはみなさない。

## 7. 語彙の学年決定アルゴリズム

### 7.1 保持すべき3種類の値

次を1つの `grade` 列へ潰さない。

1. `first_observed_grade`: 教科書資料で最初に観測された学年
2. `recommended_grade`: 本アプリで通常使用を開始する学年
3. `orthography_grade`: その漢字表記をふりがななしで使える学年

たとえば、話し言葉として1年生にも分かる語が、漢字では3年生以降となる場合がある。この語は `recommended_grade=1`, `orthography_grade=3` とし、1・2年では初出ふりがな型で使える。

### 7.2 候補生成

1. V1の `小学校低学年` 区分を1〜3年用、`小学校高学年` 区分を4〜6年用の基礎候補集合として別々に保持する。
2. V2から、1・2・3年それぞれの教科別頻度を取得する。
3. UniDicで見出し、読み、品詞、活用を正規化する。
4. 固有名詞、数詞、接辞、記号、誤解析候補を別キューへ分離する。
5. 同形異義語と多義語を、意味・品詞単位に分割する。

### 7.3 自動算出

自動処理で確定してよいのは、観測値と推奨候補までとする。

```text
first_observed_grade
  = V2で頻度が1以上となる最小学年

grade_frequency[g]
  = 学年gの全対象教科における頻度合計

grade_range[g]
  = 学年gで出現した教科数
```

`recommended_grade_candidate` は次の優先順で作る。

1. V1で低学年語彙に含まれ、V2の当該学年で複数回または複数教科に出る語
2. V1にはないが、V2で高頻度かつ複数教科に出る生活基本語
3. 国語読解に必要だが教科偏在する語
4. 低頻度、固有名詞、専門語、抽象度の高い語

頻度閾値は、実データの分布を見る前に固定しない。DB作成スクリプトが学年別の頻度分位点と対象語数をレポートし、初版の目視確認で閾値を確定する。閾値は `derivation_rule` の版として保存する。

### 7.4 開発者による目視確認

最終的な `recommended_grade` は、少なくとも次を目視確認して確定する。

- その語を耳で聞いたときに対象学年が理解できるか
- 日常生活・学校生活で経験可能か
- 文章中の文脈で意味を推測できるか
- 多義語のうち、どの意味を許可するか
- 心情語・抽象語として難しすぎないか
- 表記だけが難しい場合、初出ふりがなで解決できるか
- 家庭環境、障害、性別、文化を不必要に決め付けないか

目視確認が終わっていない語は、生成本番では使用しない。

この目視確認は語彙項目の適切さを確認するものであり、生成された本文・設問全体の公平性チェックを代替しない。問題単位の見直しは [item-review-and-release.md](item-review-and-release.md) に従う。

## 8. 漢字・ふりがな判定

### 8.1 既習漢字集合

```text
grade=1 → 第1学年配当80字
grade=2 → 第1〜2学年配当240字
grade=3 → 第1〜3学年配当440字
```

ただし `curriculumPoint=start|summer|end` を指定する場合、学年内の配当順は文部科学省から一意には得られない。学期別既習データは、使用教材または利用者が指定する別レイヤーとして管理し、国の学年配当と混ぜない。

### 8.2 表記選択

```text
語彙が対象学年より難しい
  → 語を置換する

grade=1、語彙は対象学年相当、表記に漢字がある
  → 既習判定にかかわらず、定義された範囲の初出だけふりがな付き

grade=2|3、語彙は対象学年相当、表記中の漢字がすべて既習
  → ふりがななしの漢字表記

grade=2|3、語彙は対象学年相当、表記に未習漢字がある、ruby_allowed=true
  → 定義された範囲の初出だけふりがな付き

正しい読み・送り仮名を保証できない
  → ひらがな表記または語を置換する
```

### 8.3 学年別の初出ふりがな

`ANCHOR-RUBY-Q12` と `ANCHOR-RUBY-Q18` で観察した「初出で読みを示し、再出時には児童が思い出して読む」構造を、1〜3年の各学年で利用する。1年生は漢字を含む全語を対象とし、2・3年生は未習漢字を含む語を対象とする。観察資料、権利状態、再利用範囲、実証上の限界は [reference-anchor-registry.md](reference-anchor-registry.md) に記録する。

初出判定は語彙素ではなく `form_id` と読みの組で行う。同じ語でも表記または読みが変わる場合は、再度ふりがなを付ける。

## 9. 作成パイプライン

```text
1. 出典台帳に資料、版、取得日、利用条件を登録
        ↓
2. 原本を raw 領域へ取得し、SHA-256を記録
        ↓
3. 文字コードをUTF-8へ変換し、元ファイルは変更しない
        ↓
4. 漢字・語彙・読み・頻度を出典別テーブルへ取り込む
        ↓
5. Unicode NFC、空白、記号、かなを正規化
        ↓
6. UniDicで語彙素・品詞・読み・活用の候補を付与
        ↓
7. 自動解析の差分と低信頼項目を人手確認キューへ送る
        ↓
8. 出典横断で同一語を統合し、根拠行は保持
        ↓
9. 学年別頻度と推奨学年候補を算出
        ↓
10. 教育・表記・安全性を目視確認する
        ↓
11. 検証ゲートを通す
        ↓
12. 配布可能データだけをJSONへコンパイル
        ↓
13. DB版、出典、加工内容、ライセンス表示を付けてリリース
```

## 11. 配布形式

Webアプリには、最低限次を出力する。

```text
language-db.manifest.json
kanji-grade-1.json
kanji-grade-2.json
kanji-grade-3.json
vocabulary-db.manifest.json
vocabulary-lower-elementary.json
vocabulary-upper-elementary.json
prototype-vocabulary-evidence.ts
lexeme-forms.json
```

各ファイルに `schema_version`, `database_release`, `generated_at`, `source_releases`, `derivation_rule_version` を含める。人手確認前の漢字最小版は `language-db.v0.1-kanji-candidate`、語彙候補版は `vocabulary-db.v0.1-ninjal-2009b-candidate` とし、`language-db.v1` や検証済みリリースを名乗らない。

語彙候補版はCC BY 4.0の出典データを再現可能に加工した配布候補であるが、教材生成への利用許可を意味しない。全件を `active_for_generation=false` とし、目視確認済みの許可語だけを将来の生成用DBへ昇格させる。

同じ入力、同じ原本チェックサム、同じスクリプト版からは、バイト単位で同じJSONを生成する。配列順を固定し、生成時刻は再現対象のデータ本体から分離する。候補JSONの `generated_at` は `null` とし、実行時刻が必要な場合は再現対象外の実行ログへ記録する。

### 11.1 現行generatorへの監査用投影

ブラウザへ15,485件の候補JSON全体を読み込ませず、現行 `prototype_lexicon` の語だけをビルド時に照合し、`src/generated/prototype-vocabulary-evidence.ts` を生成する。照合キーは限定語彙の表記と読みであり、低学年候補に一意に一致しない場合はビルドを失敗させる。高学年候補にしか一致しない語も失敗対象とする。

この投影は、既存の限定語彙が低学年候補母集団に根拠を持つことを検査し、生成物へ候補DB版・出典語ID・照合範囲を残すためのものである。`active_for_generation=false` の候補を文章生成の選択肢へ昇格させず、人間による意味・安全性・個別学年の確認済みとも扱わない。監査範囲は `prototype_lexicon` でタグ付けされた出現だけで、テンプレート中のその他の語句や文法は対象外とする。

## 12. 検証仕様

### 12.1 漢字

- 第1学年が80字である
- 第2学年が160字である
- 第3学年が200字である
- 1〜3年の累積が440字である
- 学年間で同じ漢字が重複しない
- すべての漢字が1コードポイントとして妥当である
- 全件がK1の出典行へ到達できる
- 配当学年と本プロジェクト推奨読みを別項目で保持する

### 12.2 語彙

- 低学年候補が6,865語である
- 高学年候補が8,620語である
- 中学校11,749語が小学校候補へ混入していない
- 原CSVと出典台帳のSHA-256が一致する
- 7語彙表の採否欄数と原資料の `出現数` が一致する
- 同一原本からバイト単位で同じ候補JSONを生成できる
- 未確認候補が `active_for_generation=true` になっていない
- 全語に安定ID、見出し、読み、品詞がある
- 全推奨学年に最低1件の根拠行がある
- Bの観測値とC/Dの推奨値が別テーブルにある
- 同形異義語を上書き統合していない
- 活用形を別の見出し語として重複登録していない
- 目視確認前の語が生成利用可能なDBに混入しない
- `prototype_lexicon` の各語が低学年候補へ表記・読みで一意に一致する
- 高学年候補にしか一致しない限定語を監査用投影へ追加するとビルドが失敗する
- 同一候補DBから監査用投影をバイト単位で再生成できる
- `license_status` が `open|approved` 以外のデータが配布物にない
- 禁止語・要文脈語の状態が欠落していない

### 12.3 表記とふりがな

- 各漢字表記に読みがある
- 送り仮名と活用が表記に一致する
- 1年生では、既習判定にかかわらず漢字を含む許可語が初出ふりがな対象になる
- 2・3年生では、選択学年の既習漢字だけなら通常表記になる
- 2・3年生では、未習漢字を含む許可語が初出ふりがな対象になる
- 同じ範囲の再出語ではふりがなが省略される
- 表記または読みが変わった場合は再度ふりがなが付く

### 12.4 学年と生成難度の独立性

本文長を含む45通りの全組合せをコーパステストする。

```text
3学年 × 5生成難度 × 3本文長 = 45条件
```

各プロファイルで、次を確認する。

- 語彙と表記は選択学年の制約に従う
- 生成難度変更だけでは語彙学年が上がらない
- 学年変更だけでは推論段階数が増えない
- 学年1・生成難度5と学年3・生成難度1を生成できる
- generator版、アルゴリズム仕様版、出題設計版、DB版、seed、テンプレート・プロンプト版、モデル設定、正規化版、ふりがな処理版を固定した再現条件を記録できる

## 14. 更新方針

- 文部科学省の学習指導要領改訂時に漢字DBを更新する
- 教科書改訂周期または語彙リスト更新時に語彙頻度を更新する
- UniDic更新は自動追随せず、差分検証してから版を上げる
- 推奨学年の変更は、変更理由と目視確認記録を残す
- 過去DB版を保持し、保存済みitem revisionの生成来歴を再確認できるようにする

## 15. 初版の作成手順

### フェーズ1：権利条件が明確な範囲

1. 文部科学省資料から1〜3年の440字を作成する
2. 漢字DBの件数・重複・出典を自動検証する
3. UniDicの採用版と修正BSD表示を固定する
4. 教育基本語彙DBの低学年・高学年候補スキーマ、取込・生成・検証スクリプトを作る
5. 語彙の意味・表記・安全性を確認する目視確認用データまたは画面を作る

### フェーズ2：語彙資料の利用確認

1. 教育基本語彙DB公式ページのCC BY 4.0表示を `source_registry` と生成物へ記録する
2. WEB版小学校教科書語彙リストの研究代表者へ利用範囲を確認する
3. V2を採用する場合だけ、確認した利用条件を `source_registry` へ記録する

### フェーズ3：語彙初版

1. V1の低学年6,865語・高学年8,620語を候補集合として取り込む
2. 語義、表記、読み、活用、安全性の確認キューを作る
3. 必要になった時点でV2の学年別観測値と個別学年の重みを追加する
4. 開発者本人が生成利用可能な語を目視確認する
5. generatorへ確認済み語彙DBを接続し、全プロファイルのコーパステストを行う
6. 完成条件を満たした後に `language-db.v1` をリリースする

## 16. 完成条件

次をすべて満たしたとき、学年別データベース仕様および初版DBを完成とする。

- 学年と生成難度プロファイルが独立した入力として定義されている
- 1〜3年の漢字440字が文部科学省資料と一致する
- 学年別語彙の観測値と推奨値が分離されている
- 全項目から出典・版・取得日・加工規則を追跡できる
- 配布データの利用条件が確認済みである
- 語彙の推奨学年を開発者本人が目視確認済みである
- ふりがな判定に語彙学年と表記学年の両方を使っている
- 15通りの学年・生成難度組合せを検証できる
- 同一原本・同一スクリプトから同一DBを再生成できる
- DB更新時に差分と理由を提示できる
- DBの語彙目視確認と、生成問題の公平性チェック・公開判断を区別している

## 17. 現時点の判断

根拠のあるデータベースは作成できる。

ただし、根拠の性質は次のように表現しなければならない。

```text
漢字の学年
  = 文部科学省が定めた配当学年

語彙の出現学年・頻度
  = 研究資料で観測された事実

語彙の推奨学年
  = 観測値と教育基本語彙をもとに、開発者本人が確認して定めた値
```

語彙の推奨学年を「文部科学省が定めた公式学年」と表示してはならない。
