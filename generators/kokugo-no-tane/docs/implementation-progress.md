## AIプロトタイプ経路

最初に動くプロトタイプを確認できるまでは、個々の成果物ごとに人間確認で作業を停止しない。AIが候補データ、実装、自動検査、ブラウザ表示を縦に接続し、次の順で自走する。

```text
漢字DB候補版
   ↓ AIによる正本照合・負例テスト
限定語彙を使う出題エンジン
   ↓ seed・正答根拠・初出ふりがな検査
ブラウザプロトタイプ
   ↓ AIによる操作・表示・印刷確認
最初の人間レビュー
```

正式な語彙DBが未完成の間は、出典と適用範囲を明示したプロトタイプ限定語彙を使用できる。ただし正式DBや学年適合性の実証済みデータとは呼ばない。

人間による漢字原典照合、作問確認、構成要素の採否は、動くプロトタイプを見ながら行う最初の人間レビューへまとめる。この延期によって、問題の状態が `unreviewed_practice`、`reviewed_practice`、`release_ready` へ進むことはない。プロトタイプは `development_preview` とし、子どもへの試行、学力判定、公開教材としての利用は人間レビュー前には行わない。

修正内容によって戻る地点を変える。

| 問題または変更 | 戻るフェーズ |
| --- | ---: |
| 語彙、学年判定、読み、権利状態の問題 | 1〜2 |
| 本文構造、設問、選択肢、正答根拠の問題 | 3〜4 |
| 表示、説明、採点基準の軽微な調整 | 5〜6 |
| 子どもの試行で作問構造の問題が判明 | 3へ戻り、4を通してから再試行 |
| 校正済み問題の本文・設問・ふりがなを変更 | 見直し・校正状態を戻し、変更内容に応じて3〜6を再実施 |

## ローカルAIプロキシの実装証拠

2026年7月15日に、AIを物語設計図の候補供給者として使うローカルライブ経路を実装し、7月16日に評価用providerをCodexヘッドレス既定へ切り替えた。

- Node.jsは公式最新リリース `26.5.0` をnodenvへ導入し、`.node-version`、`.nvmrc`、`package.json#engines` で固定した。
- `server/ai-proxy.ts`：loopback限定HTTP、CORS allowlist、入力上限、レート制限、タイムアウト、再試行、キャッシュ、安全なエラー写像を実装した。
- `server/openai-story-plan.ts`：Responses API、`store:false`、Structured Outputs、`reasoning.effort=high`、`story-plan-prompt.v2` を実装した。

- `server/story-plan-context.ts`：学年までの配当漢字候補、教育基本語彙候補DBの版・件数・生成利用不可状態、限定語彙への監査投影だけが接続済みであること、参照アンカー12・18の抽象的読み支援を `story-plan-context.v1` として構成した。
