## 1. 目的

本書は、生成難度プロファイルと児童にとっての実測難易度を区別し、児童試行、項目分析、記述採点の一致、公平性、校正の失効条件を定める。

本文長、根拠間距離、推論段階などは難しさの予測因子であり、実際の難易度そのものではない。児童データを取得していない問題へ、実測済みの難易度ラベルを付けない。

本書は日常練習用の自動生成を禁止するものではない。未校閲・未校正であることを明示した練習素材と、実測難易度を表示する固定問題を区別する。

## 2. 用語

| 用語 | 意味 |
| --- | --- |
| 生成難度プロファイル | 現行プロトタイプでは各設問セットの心情記述問題の根拠配置を制御する1〜5の生成条件 |
| 本文長 | 短め・ふつう・長めから独立して選ぶ生成条件 |
| 実測難易度 | 固定された問題版を、定義された対象児童・実施条件で試行した結果 |
| 問題版 | `item_id` と `item_revision` で一意に定まる本文、設問、表示、採点基準の組 |
| 有効回答 | 対象条件を満たし、欠測・中断・重複などの除外条件に該当しない回答 |
| 校正 | 正答率、識別力、無回答、誤答分布などから問題の性質を確定すること |
| 公平性 | 読解能力以外の背景や属性が、不必要に得点へ影響しないこと |

## 3. 評価段階

### 3.1 生成品質コーパス

同一条件で異なるseedを使い、最低30セットを生成して、正答不能率、重複率、語彙違反などを調べる。これはシステムの生成品質検査であり、児童試行でも実測難易度の校正でもない。

### 3.2 認知・操作性試行

対象学年の児童に、設問の意味、回答操作、ふりがなの見え方、読み止まりを確認する。

- 目安：各対象学年8〜15人
- 目的：誤解、操作不能、強い負荷、表示上の問題の発見
- 結果：改善仮説を作るために使い、難易度ラベルは付けない

### 3.3 暫定校正

同じ問題版、同じ対象学年、同等の実施条件で、1問あたり100件以上の有効回答を集める。

100件未満でも傾向は報告できるが、`provisionally_calibrated` を付けない。100件という値は本プロジェクトの初期運用基準であり、標準化テストとしての十分性を保証しない。

### 3.4 安定校正

1問あたり300件以上の有効回答を集め、異なる実施日または集団でも指標が再現することを確認する。安定校正前は、高い利害を伴う判断へ使用しない。

## 4. 対象児童と実施条件

問題版ごとに、次を固定または記録する。

- 学年
- 学期・実施月・`curriculum_point`
- 学校内外、個別・集団などの実施環境
- 紙、PC、タブレットなどの提示媒体
- 縦書き・横書き、文字サイズ、行間、ふりがな表示
- 読み上げの有無と範囲
- 制限時間の有無と実時間
- 本文を見返せるか
- 設問順、選択肢順
- 練習問題や事前説明
- 回答方法、入力補助、代替回答
- 中断、欠測、重複回答の除外規則

小学1年生では、同じ学年でも実施時期による差が大きい。`grade=1` だけで集計せず、少なくとも `start|summer|end` または実施月を保持する。

読み上げを使用した結果は、文字を読む負荷を含む通常条件と混ぜない。代替回答を認めた場合は、記述技能の影響を除いた条件として別に集計する。

## 5. 収集する指標

### 5.1 全形式共通

| 指標 | 定義 |
| --- | --- |
| 正答率 | 完全正答数 ÷ 有効回答数 |
| 部分正答率 | 部分点を得た回答数 ÷ 有効回答数 |
| 無回答率 | 無回答数 ÷ 対象回答数 |
| 平均所要時間 | 極端値処理後の回答時間 |
| 中央所要時間 | 回答時間の中央値 |
| 修正率 | 一度入力後に回答を変更した割合 |
| 識別力 | 当該問題得点と、当該問題を除いたセット得点との関連 |

4問だけのセットでは総得点の範囲が狭く、識別力や信頼性係数が不安定になりやすい。複数セットを共通尺度で実施するか、指標の信頼区間を併記する。4問1セットの係数だけで良問と断定しない。

### 5.2 選択問題

- 選択肢別の選択率
- 正答選択率
- 各誤答が想定した誤読に対応したか
- どの児童にも選ばれない誤答の有無
- 選択肢位置による偏り

有効回答100件以上で、どの誤答も5％未満しか選ばれない場合は、誤答として機能していない可能性をレビューする。5％未満だけを理由に自動却下せず、意図した誤読と回答内容を確認する。

### 5.3 抜き出し・記述問題

- 完全正答、部分正答、誤答、無回答の比率
- 採点要素ごとの達成率
- 許容例にないが妥当な回答の比率
- 誤字、表記ゆれ、助詞不足による減点件数
- 同じ回答を別セッションで再採点したときの完全一致率
- 再採点のCohenのκまたは順序尺度に応じた重み付きκ

暫定校正では、回答の少なくとも20％かつ30件以上を抽出し、開発者本人が最初の採点結果を見ずに別セッションで再採点する。目標は完全一致率90％以上かつκ0.80以上とする。どちらかを下回る場合、採点基準を修正して再採点する。別の大人に採点を頼める場合は、自分の採点との差も追加確認する。

## 6. 実測難易度

### 6.1 表示ラベル

生成難度1〜5を、そのまま実測難易度1〜5として表示しない。暫定校正後は、対象学年・実施条件を付けて次の記述ラベルを使用する。

| 正答率 | 暫定ラベル |
| ---: | --- |
| 0.91以上 | とても取り組みやすい |
| 0.75〜0.90 | 取り組みやすい |
| 0.50〜0.74 | 標準的 |
| 0.30〜0.49 | むずかしめ |
| 0.29以下 | とてもむずかしい |

このラベルは能力尺度ではなく、当該集団・条件での正答率の記述である。正答率の信頼区間、対象学年、実施時期、回答数を併記する。

### 6.2 採否の初期基準

標準的な練習・評価問題候補として、次を初期基準とする。

- 有効回答100件以上
- 正答率0.30〜0.90
- 無回答率：選択問題10％以下、記述問題20％以下
- 修正済み項目全体得点との識別指標0.20以上
- 選択問題の正答が統計上も一意に機能している
- 記述採点の完全一致率90％以上かつκ0.80以上

基準外でも、導入問題や挑戦問題など目的が明確なら直ちに却下しない。利用目的を出題設計へ追加し、開発者本人が例外理由を記録する。

## 7. 生成難度の校正

生成難度プロファイルごとに、学年・実施時期別の正答率、所要時間、無回答率を集計する。

次を確認する。

- 生成難度を上げると、平均的には正答率が下がるか。
- 生成難度を上げると、各設問セットの心情記述問題の根拠間距離が意図どおり変化するか。
- 本文長を変えると、他の条件を固定したまま文字数帯と所要時間が意図どおり変化するか。
- 学年を変えただけで推論負荷が変化していないか。
- 学年1・生成難度5が、学年外語彙の増加で難しくなっていないか。
- 学年3・生成難度1が、語彙を不自然に幼くすることで易しくなっていないか。

順序が再現しない場合は、問題のラベルを手作業で合わせるのではなく、生成パラメータと出題設計を見直す。見直し後は `generation_profile_version` を上げ、旧版と混ぜて集計しない。

## 8. 公平性評価

### 8.1 内容チェック

児童試行前に、[item-review-and-release.md](item-review-and-release.md) の公平性・アクセシビリティ確認を完了する。特に次を記録する。

- 家族構成、経済環境、地域、文化経験の前提
- 性別と性格・役割の固定化
- 障害、病気、発達特性の描写
- 専門知識や特定の遊び・行事の経験
- ふりがな、字形、行間、縦横書きによる読みやすさ
- 不快感、恐怖、羞恥を生み得る題材

### 8.2 集団別分析

収集と利用に適法な根拠があり、本人・保護者の同意と十分な件数がある属性だけを分析する。個人を特定できる小集団を作らない。

比較候補は次のとおり。

- 実施時期
- 提示媒体
- 読み上げ・入力補助の有無
- 日本語学習歴
- 読みの支援ニーズ

各群50件未満の比較は探索的と明記し、差がないことの証明に使わない。素の正答率差だけで公平・不公平を決めず、全体の読解得点または共通問題で能力水準をそろえた分析を行う。

能力水準を調整した後も15ポイント以上の正答率差が残る、または統計的な差異項目機能が示唆される場合は、`fairness_recheck_required` として内容を再確認する。統計差だけで自動却下せず、本文、語彙、題材、表示、採点のどこに原因があるかを確認する。

### 8.3 アクセシビリティ評価

次を別々に確認する。

- 初出ふりがなが読みを助けたか、視線移動を増やしたか
- 同じ語の再出でふりがなを省いたとき、読みに戻れるか
- 文字サイズ、行間、ルビサイズが読み止まりに影響したか
- 縦書きと横書きで所要時間や誤読が変わるか
- キーボード入力や手書きが記述得点へ影響したか

問題12・18から得た初出ふりがなの仮説は [reference-anchor-registry.md](reference-anchor-registry.md) に記録する。教材で観察した設計を、そのまま全児童に有効な事実とみなさず、試行で確認する。

## 9. 校正を失効させる変更

| 変更 | 失効するもの | 再実施 |
| --- | --- | --- |
| 本文の意味、出来事、根拠配置 | 全校正 | 認知試行からやり直す |
| 設問、選択肢、正答、採点要素 | 全校正 | 認知試行からやり直す |
| 対象学年、学期 | 当該対象への一般化 | 新しい対象で校正する |
| 語彙・漢字・ふりがな | 正答率、所要時間、アクセシビリティ | 少なくとも暫定校正をやり直す |
| 文字サイズ、行間、縦横書き、媒体 | 所要時間とアクセシビリティ | 同等性試行を行う |
| 誤字修正のみ | 原則維持 | 別セッションで意味・表示に影響しないことを再確認する |
| 採点表現の明確化で得点境界が不変 | 採点一致 | 答えを隠した再採点を実施する |

生成モデル、プロンプト、アルゴリズム、DB版を変更しても、保存済みの問題版の本文・表示・採点が完全に同一なら、その固定版の校正は直ちには失効しない。ただし、新たに生成した問題へ旧版の校正を引き継がない。

## 10. データと個人情報

- 児童の氏名を分析IDとして使わない。
- 必要最小限の属性だけを、利用目的と保存期間を明示して収集する。
- 保護者同意、本人への説明、撤回方法を用意する。
- 公開レポートでは少人数群を統合または非表示にする。
- 自由記述に個人情報が含まれる可能性を考慮し、アクセスを制限する。
- 原回答、採点履歴、除外理由を追跡可能にしつつ、公開用データは匿名化する。

## 12. 完成条件

校正済みと表示するには、次をすべて満たす。

- 問題版と実施条件が固定されている。
- 対象学年、実施時期、有効回答数、除外規則を記録している。
- 正答率、無回答率、選択肢分布または採点要素分布を計算している。
- 記述問題は別セッションで再採点し、一致度を確認している。
- 生成難度と実測難易度を別項目で保持している。
- 公平性とアクセシビリティを内容面・データ面の両方から確認している。
- 標本不足の集団について「差がない」と結論していない。
- 変更による校正失効を管理できる。
- 個人情報と同意を適切に管理している。
- 最終的な公開判断が [item-review-and-release.md](item-review-and-release.md) に従っている。
