## 1. 目的

本書は、作問設計の参考にした既存問題を、外部文脈がなくても識別できる参照IDとして管理する。

参照問題から抽出するのは、測定能力、根拠配置、設問形式、読み支援などの抽象的特徴だけである。本文、人物、舞台、固有の出来事、展開、選択肢を複製・言い換えして生成素材にしない。

## 2. 権利と取扱い

- 元PDFは家庭内資料としてリポジトリ外に保管する。
- 元PDF、ページ画像、全文OCR、復元可能な本文データをリポジトリへ追加しない。
- 本書には、設計判断を再現するために必要な書誌・位置・抽象的観察だけを記録する。
- 出版・転載・学習用DBへの同梱条件は未確認であるため、`rights_status=analysis_only` とする。
- 参照問題との類似度検査に元本文を使う場合も、公開配布物へ原文または復元可能な特徴量を含めない。

いずれも児童データによる難易度・有効性の実証ではない。アンカーは生成難度や実測難易度の値を直接決めない。

### 4.1 `ANCHOR-PRINT-ANSWER-SHEET` から抽出する紙面原則

- 読む順序と書く位置を、右から左への一貫した列構成で示す。
- 指定字数、選択式、抜き出し、理由付き記述に同じ解答欄を使い回さない。
- 指定字数問題だけに1文字単位のマスを使い、字数未指定の抜き出しと長い記述には文字区切りのない細長い縦書き欄を使う。
- 外枠、設問番号、ページ番号、十分な余白で、紙面のまとまりを示す。
- 問題本文より罫線を弱くし、児童の鉛筆記入を主役にする。

再利用するのは以上の抽象原則だけである。元資料の正確な寸法、罫線位置、装飾、解答例の赤字、問題文、画像を複製しない。

## 5. `ANCHOR-STORY-Q06`

### 5.1 所在

| 項目 | 内容 |
| --- | --- |
| `source_id` | `SRC-JP-G1-SUMMER-HOMEWORK-2026` |
| 問題番号 | 6 |
| 本文 | 冊子ページ6〜7、PDFページ2の上段左 |
| 設問・解答例 | 冊子ページ8〜9、PDFページ2の下段左 |
| 本文末の出典表示 | 井上よう子『くじゃくのジャックのだいだっそう』文研出版より、と紙面に表示 |

### 5.2 抽出した測定設計

紙面には、1つの物語本文に対し、次の異なる反応を求める設問が組み合わされている。

- 人物を表す明示語の指定字数抜き出し
- 特定場面の人物の気持ちを選ぶ問題
- 人物や集団の好みとして明示された事実を本文から特定する問題
- 複数の出来事や行動を踏まえて気持ちを説明する問題

この組合せは、[item-blueprint.md](item-blueprint.md) の標準4問を設計する際の定性アンカーとする。ただし、明示された性格語の抜き出しは人物像推論ではなく、主に明示情報探索として扱う。

### 5.3 再利用してよい特徴

- 1つの本文で、明示情報探索と心情推論を分けて問う。
- 抜き出し問題は字数条件と本文上の一意性を対応させる。
- 心情推論には、出来事と人物の反応の両方を根拠として残す。
- 4問が同じ一文だけで解けないよう、根拠を分散する。

### 5.4 模倣してはいけない特徴

- 動物園、クジャク、見栄、餌、脱走に関する人物・舞台・出来事
- 原文の会話、語順、展開、結末
- 選択肢、指定字数、正答表現の直接的な転用

### 5.5 難度の扱い

このアンカーを「生成難度3」または「小学1年生の標準難易度」と確定する実測根拠はない。本文長や根拠配置を生成パラメータの参考にはできるが、難度値は [calibration-and-fairness.md](calibration-and-fairness.md) に従って校正する。

## 6. `ANCHOR-RUBY-Q12`

### 6.1 所在

| 項目 | 内容 |
| --- | --- |
| `source_id` | `SRC-JP-G1-SUMMER-HOMEWORK-2026` |
| 問題番号 | 12 |
| 本文 | 冊子ページ14〜15、PDFページ3の上段左 |
| 設問・解答例 | 冊子ページ16〜17、PDFページ3の下段左 |

### 6.2 観察した特徴

- 小学1年生が耳で理解できる物語を、すべてひらがなへ崩さず漢字を交えて表記している。
- 読みの支援が必要な漢字には小さな読みが添えられている。
- 読み支援は本文の流れの中に置かれ、語の意味説明や脚注へ読者を移動させない。
- 同一語を再び読む際に、先の読み支援を手掛かりとして利用させる構造が見られる。

### 6.3 本プロジェクトへ採用する仮説

```text
語彙は対象学年で理解できる
かつ
漢字表記だけが未習である
    ↓
本文内の初出で読みを示す
    ↓
同じ語・同じ読みの再出では、読みを省いて思い出して読ませる
```

これは教材観察から得た設計仮説であり、読みやすさや学習効果が実証済みという意味ではない。ふりがな密度、視認性、再出時の読め方は児童試行で確認する。

### 6.4 模倣してはいけない特徴

- 親子の約束、運動会、競走に関する固有の展開
- 原文の人物関係、会話、心情変化
- 設問文、正答例、紙面配置の直接的な転用

## 7. `ANCHOR-RUBY-Q18`

### 7.1 所在

| 項目 | 内容 |
| --- | --- |
| `source_id` | `SRC-JP-G1-SUMMER-HOMEWORK-2026` |
| 問題番号 | 18 |
| 本文 | 冊子ページ22〜23、PDFページ4の上段左 |
| 続く設問・解答例 | 冊子ページ24、PDFページ4の下段左 |

### 7.2 確認できる初出・再出

本文では、冒頭付近の「学校」に読みが添えられ、その後に同じ表記が現れる箇所では読みが省かれている。また、「二人」も初出側に読みがあり、後続の同一表記では省略される配置を確認できる。

この観察から、初出判定を漢字1文字単位ではなく、語と読みの組で管理する必要があると判断する。同じ表記でも読みや意味が異なる場合は、別の初出として扱う。

### 7.3 再利用してよい特徴

- 本文を1つの初出判定範囲として扱う。
- 年齢相当語彙で、表記だけが未習の場合に読みを支援する。
- 同じ語・同じ読みの再出では、読み支援を機械的に繰り返さない。
- 語彙ID、表記ID、読み、出現順、ふりがな表示理由を保持する。

題名、設問、選択肢、解答で初出判定をリセットする規則は、この参照問題から直接得たものではなく、本プロジェクトが単独可読性のために追加した設計判断である。参照由来の事実とプロジェクト判断を混同しない。

### 7.4 模倣してはいけない特徴

- 友人関係、けんか、仲直りに関する固有の人物・展開
- 原文の会話、行動、場面順
- 設問文、正答例、紙面配置の直接的な転用

## 8. 設問形式アンカー

問題6、12、18から抽出した設問形式は [question-pattern-expansion.md](question-pattern-expansion.md) で設計案として整理する。

- `ANCHOR-STORY-Q06` は、指定字数抜き出し、明示心情の選択、明示事実の抜き出し、行動からの心情記述を組み合わせる基準構成として扱う。
- `ANCHOR-QUESTION-FORM-Q12` は、1つの物語で異なる根拠操作と回答形式を6問へ配分する構成を参考にする。
- `ANCHOR-QUESTION-FORM-Q18` は、抽象表現の具体化、語の働き、行動からの心情推論を組み合わせる構成を参考にする。

再利用してよいのは、設問が要求する認知操作、根拠の種類、回答形式、解答欄の抽象的要件だけである。元資料の人物、出来事、文章、選択肢、正答、設問文、紙面寸法を複製しない。

## 9. アンカーを追加するとき

新しいアンカーには、少なくとも次を記録する。

```text
anchor_id
source_id
資料名
問題番号・ページ・PDF内位置
権利状態
確認日・確認方法
抽出した測定能力
本文・設問・表記の抽象的特徴
再利用してよい特徴
模倣してはいけない表層要素
難度判断の根拠と限界
```

元資料を保存できない場合も、位置、ハッシュ、分析結果を残す。出典や権利状態が分からないアンカーは生成仕様の根拠へ昇格させない。

## 10. 完成条件

- 「問題6相当」「問題12番・18番」だけの属人的な参照を残していない。
- すべての参照がsource IDとアンカーIDで追跡できる。
- 元資料の権利状態と公開可否を記録している。
- 参照本文をリポジトリへ転載していない。
- 抽象的に再利用する特徴と、模倣禁止の表層要素を区別している。
- 定性的アンカーを実測難易度の証拠として扱っていない。
