# 数字の階段ジェネレータ 仕様書

## 目的

`数字の階段` の印刷用教材をブラウザだけで生成する。Web上で解答を入力するアプリではなく、問題・解答・IDマップ・確定順・解き方を生成し、印刷またはPDF保存するための静的ページとする。

## 配置

- `index.html`: 画面構造。
- `styles.css`: 画面表示と印刷用スタイル。
- `js/problem-generator.js`: 出題エンジン。問題テンプレート、乱数、初期配置探索、候補検査の呼び出し。
- `js/solver.js`: ソルバーエンジン。分岐なし伝播、整数範囲ソルバー、一意解判定、メトリクス生成。
- `js/renderer.js`: SVG描画、確定順HTML、解き方HTML。
- `js/app.js`: DOM操作、レベル選択、モード選択、印刷ボタン。
- `server.sh`: ローカルHTTPサーバー起動スクリプト。任意のポート番号を引数で受け取り、既定ブラウザを開く。

外部ライブラリとビルド処理は使わない。ブラウザJSはES Modulesで書くため、`file://` で直接開くのではなく、GitHub PagesまたはローカルHTTPサーバーで `web/` 配下を配信して動かす。

## 起動方法

```bash
cd web
./server.sh
./server.sh 9000
```

## ルール

線でつながる3マス `(a, b, c)` は等差数列として扱う。

```text
a + c = 2 * b
```

- `b` は両端 `a`, `c` のちょうど真ん中。
- 3マスのうち2マスが分かれば、残り1マスが決まる。
- この確定を繰り返して全マスが埋まる問題だけを採用する。
- 初期配置で最初に打てる手は1箇所だけにする。
- 標準モードでは、途中で複数の確定候補が出ることを許す。
- 一本道モードでは、全ステップで確定候補が1箇所だけの問題だけを採用する。
- レベル6は形の性質上、一本道モードの候補を用意していない。選択された場合は、白紙にせず理由とエラーメッセージを表示する。
- レベル7は初期状態の確定候補が0箇所の本の問題として扱う。通常の伝播ではなく、1〜20の整数範囲と式整理で一意解を確認する。

## レベル

レベル1〜7を用意する。レベル1〜6は既存の宿題モデルから形を借り、数値と初期配置を生成時に変える。レベル7は本の問題を固定値で再現する。

| レベル | 形の元 | 意図 |
|---|---|---|
| 1 | `problem1` | 横長の基本連鎖 |
| 2 | `problem2` | 折れ曲がりのある連鎖 |
| 3 | `problem3` | 複数ブロックの連鎖 |
| 4 | `problem4` | 斜めに下りる階段型の連鎖 |
| 5 | `problem5` | 広い盤面で左右に伸びる連鎖 |
| 6 | `problem6` | 大きい盤面で探索量が多い連鎖 |
| 7 | `book-problem` | 初手なし。整数範囲と式整理で一意に決める本の問題 |

## 生成手順

1. 選択されたレベルのテンプレートを読む。
2. テンプレートの正解値にランダムな整数変換をかける。
   - 例: `新しい値 = 元の値 * scale + offset`
   - ルールの `a + c = 2 * b` はこの変換後も保たれる。
3. 初期配置は、各レベルの作問済みパターンを種にして探索生成する。
   - まず、元の宿題から起こした基準パターンを読む。
   - 初期値の数は、各レベルの `minGivens` を下限として探索する。
   - 初期値マスと空欄マスを入れ替える変異を繰り返す。
   - 変異後の候補は、初手候補が1箇所だけ、最初から完成している3マス組がない、分岐なし推論で全マスが確定、線形制約として一意解、正解値と一致、の5条件を満たすものだけ採用する。
   - さらに、抜いても解ける余分な初期値がある候補は採用しない。
   - 一本道モードでは、全ステップの `ready` 候補数が1であることも採用条件に加える。
4. 採用済み候補の中から、seedに基づいて1つを選ぶ。
   - 同じseedなら同じ問題になる。
   - seedを変えると、数値だけでなく初期配置パターンも変わる。
5. 出題エンジンは候補盤面をソルバーエンジンに渡し、`uniqueSolution`, `maxReadyCount`, `givenCount`, `stepCount`, `patternSignature` などのメトリクスを受け取る。

## 出力

レベルを選ぶと、以下を同時に生成する。

1. `問題`
   - 初期値だけを表示する。
   - 未確定マスは空欄にする。
2. `解答`
   - 全マスの正解を表示する。
3. `IDマップ`
   - 全マスに `t0`, `m0` などのIDを表示する。
   - 確定順や解き方と照合するための地図として使う。
4. `あてずっぽう無しの確定順`
   - `(t0, m0, b1)` のような3マス単位と、式を順番に表示する。
5. `解き方`
   - 初手が1箇所だけであること、初手の見つけ方、その後の進め方を日本語の箇条書きで表示する。

## 印刷設計

- UIの操作部分は印刷時に非表示にする。
- `問題`, `解答`, `IDマップ` はそれぞれA4縦の紙面として印刷しやすい比率で表示する。
- `あてずっぽう無しの確定順` と `解き方` は通常のHTMLとして印刷できるようにする。

## 完了条件

- `web/` をHTTP配信して `index.html` を開くと動く。
- レベル1〜7を選ぶと問題が生成される。
- 問題、解答、IDマップ、確定順、解き方が表示される。
- 生成された問題は分岐なし推論で全マスが確定する。
- 生成された問題の初手候補は1箇所だけである。
- 印刷またはブラウザのPDF保存に適した画面になっている。
